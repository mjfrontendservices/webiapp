Progressive Web App

Progressive web apps are type of applications that lives on the web but functions more and more like a normal mobile or desktop applications. This apps are not deployed on any apps stores or apple stores, instead it is deployed the same as a normal website. Here are some advantages of PWA.

1. Cross platform – works on multiple platforms (Android, iOS) and devices (mobile and desktop computers).

2. Installation – users don't need an app store to install your app.

3. App like experience – your web app works same as a normal mobile or desktop app.

4. Offline capabilities – PWA Native has the ability to work offline.

5. Cheap – PWA are cheaper to develop than normal apps.

6. Easy to develop – uses web standard technologies for development.

Software Development Life Cycle

Webiapp follows a software development life cycle (SDLC) in web app development. This includes: Planning, Analyzing Requirements, Design, Implementation, Testing, Deployment and Maintenance.


1. Planning

The first step is planning the mobile web app functionality and gathering all the necessary features and requirements of the app. This includes

* Making an outline of features and requirements.

* Estimating cost

* Gathering the relevant information that can be added to the app.

2. Analyzing Requirements

After the features and requirements has been outlined, its time to analyzed each requirements. This includes listings all the needed information of the specific requirement.

Example:
FEATURE : User Registration
> Username input field.
> Password input field.
> Checkbox – Accept terms and conditions.
> Submit button.
> Save user in the database.

3. Design

After the analyzing all the requirements and features, its time to design your mobile web app. This includes designing the user interface, responsive designs, color schemes, image positions, logo, font styles, programming languages to be used, frameworks, relational database relationship design, application architecture, mobile aspects, supported browsers, and more.

4. Implementation/Coding

After the design, its time for the implementation and coding. This includes writing codes using the chosen programming language or framework. Designers will continue designing the user interface and asking for client approval. Testers will test the mobile web app if the requirement is achieve or not, continue looking for bugs and flaws on the app and making test cases.

5. Testing

After the mobile web app has been built and before deploying it on the remote server, it will undergo deep testing. This is to make sure that the application requirements are achieved, design are achieved, previous bugs and errors are fixed. Looking for extra bugs to fix and making sure that the application flows according to plan and functionalities are working as expected.

6. Deployment

After the mobile web app is built, tested and perfectly created, its time for deploying it on the remote server for production. This includes setting up the domain name, setting up the database, setting up the host and making the mobile web app live and accessible to all users.

7. Maintenance

The final step is maintenance. After the mobile web app is deployed, maintenance and client support is the next. This includes educating the client on the flow of the app such as exploring the admin dashboard, and more. Also gathering the users' feedback about the application.
